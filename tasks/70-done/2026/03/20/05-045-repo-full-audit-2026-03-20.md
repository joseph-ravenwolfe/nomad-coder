## Findings

### 1. HIGH — Docker base image security findings

Editor diagnostics report high vulnerabilities for every `node:24-slim` stage in `Dockerfile`:

- `Dockerfile:20` (`FROM node:24-slim AS deps`)
- `Dockerfile:34` (`FROM node:24-slim AS build`)
- `Dockerfile:52` (`FROM node:24-slim AS runtime`)

Even with runtime `apt-get upgrade`, scanner still reports six high vulnerabilities.

**Recommended remediation:**

- Re-scan against a pinned digest and/or newer patched base image.
- Consider minimizing attack surface further (distroless runtime stage or slimmer runtime dependency footprint).
- Add a CI container scan gate with explicit policy thresholds.

### 4. LOW — README container version tag is stale

**Recommended remediation:**

### 5. LOW — Docker reproducibility risk from floating pnpm bootstrap

`Dockerfile` uses `corepack prepare pnpm@latest --activate` in build stages.

This can cause non-reproducible builds as `latest` moves.

**Recommended remediation:**

- Pin pnpm version in Dockerfile to the same major/minor used by the repo (or exact version).
