# [4.3.1] — 2026-03-20

- **Task Runner agent** (`.github/agents/task-runner.agent.md`) — third-tier agent for subagent delegation; Overseer spawns Task Runners for scoped implementation tasks when no Worker sessions are active
- **Three-tier task strategy** — investigation (read-only), direct (staged), and worktree (branched) modes documented in `tasks/worktree-workflow.md`; includes claim workflow with baseline snapshots
- **`tasks/claim.ps1`** — script for claiming queued tasks; stages a baseline snapshot of the task file before moving to `3-in-progress/`

- **Agent hierarchy refined** — Overseer → Worker → Task Runner delegation chain with clear rules for each tier; worker and task-runner agent files updated with worktree isolation, task board movement, and idle notification rules
