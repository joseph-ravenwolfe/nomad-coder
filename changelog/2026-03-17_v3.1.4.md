# [3.1.4] — 2026-03-17

## Changed

- Changed `dequeue_update` default timeout from 60 s to 300 s (maximum), optimized for agent listen loops
- Renamed `empty: true` response to `timed_out: true` for blocking timeouts; `empty: true` is now exclusive to instant polls (`timeout: 0`)
- Updated `docs/behavior.md`, `docs/communication.md`, `docs/design.md`, and `docs/setup.md` to reflect new default, response semantics, and the idle check-in rule
