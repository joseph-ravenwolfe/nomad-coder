# v5.0.0 — 2026-04-03

## Breaking Changes

### Identity Token Redesign — Single Integer Auth

The `identity: [sid, pin]` tuple parameter has been **removed** from all 50+
MCP tools and replaced with a single integer `token`.

**Migration:**

```diff
- { identity: [sid, pin], ... }
+ { token: sid * 1_000_000 + pin, ... }
```

The `token` value is returned directly by `session_start`:

```diff
- const { sid, pin } = await session_start({ name: "Worker" });
+ const { token } = await session_start({ name: "Worker" });
  // pass token to all subsequent tool calls
```

**Motivation:**

- Eliminates string-coercion workarounds introduced in task 10-178
- Single integer is natively supported by all MCP JSON Schema validators
- Simpler agent code — no array destructuring, no two-value state
- No semantic change: `token = sid * 1_000_000 + pin`; decoding is lossless

**Scope of change:**

- All 50+ tool schemas: `identity: array` removed, `token: integer` added
- `session_start` returns `{ token, sid, pin, ... }` (backward-compatible addition)
- New central `decodeToken(token)` function in `src/tools/identity-schema.ts`
- New `src/session-gate.ts` with `requireAuth(token)` helper
- All test files updated to use `token` parameter
- Documentation updated (AGENTS.md, README.md)
